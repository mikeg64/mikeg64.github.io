\doxysubsection{Detailed Description}
Controls output of data. 

P\+U\+R\+P\+O\+SE\+: Controls output of data. Output is divided into three types\+:
\begin{DoxyItemize}
\item 1. dump\+\_\+$\ast$()\+: A\+LL variables are written in $\ast$ format over W\+H\+O\+LE grid
\item 2. output\+\_\+$\ast$()\+: O\+NE variable is written in $\ast$ format with various options
\item 3. restarts\+: special form of a dump, includes extra data The number and types of outputs are all controlled by $<$ouput\+N$>$ blocks in the input files, parsed by the functions in \mbox{\hyperlink{par_8c}{par.\+c}}. ~\newline

\end{DoxyItemize}

T\+O\+T\+AL N\+U\+M\+B\+ER of outputs is controlled by \textquotesingle{}maxout\textquotesingle{} in $<$job$>$ block in input file. Only the first \textquotesingle{}maxout\textquotesingle{} $<$output\+N$>$ blocks are processed, where N $<$ maxout. If N $>$ maxout, that $<$output\+N$>$ block is ignored.

O\+P\+T\+I\+O\+NS available in an $<$output\+N$>$ block are\+:
\begin{DoxyItemize}
\item out = cons,prim,d,M1,M2,M3,E,B1c,B2c,B3c,ME,V1,V2,V3,P,S,cs2,G
\item out\+\_\+fmt = bin,hst,tab,rst,vtk,pdf,pgm,ppm
\item dat\+\_\+fmt = format string used to write tabular output (e.\+g. \%12.\+5e)
\item dt = problem time between outputs
\item time = time of next output (useful for restarts)
\item id = any string
\item dmin/dmax = max/min applied to all outputs
\item palette = rainbow,jh\+\_\+colors,idl1,idl2,step8,step32,heat
\item x1,x2,x3 = range over which data is averaged or sliced; see parse\+\_\+slice()
\item usr\+\_\+expr\+\_\+flag = 1 for user-\/defined expression (defined in \mbox{\hyperlink{problem_8c}{problem.\+c}})
\item level,domain = integer indices of level and domain to be output with S\+MR
\end{DoxyItemize}

E\+X\+A\+M\+P\+LE of an $<$output\+N$>$ block for a V\+TK dump\+:
\begin{DoxyItemize}
\item $<$output1$>$
\item out\+\_\+fmt = vtk
\item out\+\_\+dt = 0.\+1
\end{DoxyItemize}

E\+X\+A\+M\+P\+LE of an $<$output\+N$>$ block for a ppm image of a x1-\/x2 slice with data averaged over 0.\+5-\/10 in x3 in ppm format\+:
\begin{DoxyItemize}
\item $<$output5$>$
\item out\+\_\+fmt = ppm
\item dt = 100.\+0
\item out = d
\item id = d
\item x3 = 0.\+5\+:10.\+0
\item dmin = 0.\+25
\item dmax = 2.\+9
\item palette = rainbow
\end{DoxyItemize}

E\+X\+A\+M\+P\+LE of an $<$output\+N$>$ block for restarts\+:
\begin{DoxyItemize}
\item $<$ouput3$>$
\item out\+\_\+fmt = rst
\item out\+\_\+dt = 1.\+0
\end{DoxyItemize}

C\+O\+N\+T\+R\+OL of output proceeds as follows\+: -\/\mbox{\hyperlink{output_8c_a9034e0a0c277e481597e9a0709908edb}{init\+\_\+output()}}\+: called by \mbox{\hyperlink{main_8c_a0ddf1224851353fc92bfbff6f499fa97}{main()}}, parses the first maxout output blocks. The info in each block is stored in an element of a global array of \char`\"{}\+Output\+\_\+s\char`\"{} structures, including a pointer to the appropriate output function. -\/\mbox{\hyperlink{output_8c_a292f4e4bd2a42e78dfe5b86da556188c}{data\+\_\+output()}}\+: called in main loop, compares integration time with time for output for each element in Output array, and calls output functions.

To add permanently a new type of output X, write a new function output\+\_\+X, modify \mbox{\hyperlink{output_8c_a9034e0a0c277e481597e9a0709908edb}{init\+\_\+output()}} to set the output function pointer when out\+\_\+fmt=X in the input file (see below for examples of pgm, ppm, etc.)

See Users Manual to add a problem-\/specific user-\/defined output function in the problem definition file.
